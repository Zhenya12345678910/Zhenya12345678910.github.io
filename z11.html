Список заданий

Для решения этого задания могут понадобится данные формулы:

N = 2^i
Где:
N — количество кодируемых равномерным кодированием объектов.
i — количество бит, которое приходится на один объект.
I = N*K
Где:
I — объём сообщения.
N — количество передаваемых номеров групп.
K — количество бит на один номер.
I = M · log2N
Где:
N — количество символов алфавита.
М — количество символов в записи сообщения.
I = log2 N
Где:
N — количество событий (символов алфавита). I — информационный объём одного символа.

Алгоритм решения:

    1. Читаем задание и выделяем то, что нам дано и что надо найти.
    2. Подбираем формулы, которые подойдут для решения задания.
    3. Вычесляем и записываем ответ.

Пример ДЕМО-2025:

    1. В номере будут содердаться десятичные цифры N = 10, 52 латинские буквы N = 52 и специальный алфавит 963-символьный N = 963. Следовательно N = 10+52+963=1025 бит(мощность нашего алфавита)
    2. По формуле N = 2**i, i = 11 бит
    3. Нам известно, что для хранение 2000 номеров отведено 693 Кбайта, сначала для удобства мы переводим в байты, следовательно 693 Кбайт = 693*1024 байт.
    4. Далее чтобы узнать сколько весит один номер, нужно I=693*1024/2000=354.816, мы округляем это число в меньшую сторону потому что для хранение отведено не более 692 Кбайт, поэтому I = 354 байт.
    5. Воспользуемся формулой k = I/i. Сначала переведем 354 байт в бит, получается I = 354*8 = 2832 бит. По формуле k = 2832/11=257.4747, округляем в меньшую сторону и получается k = 257.
    6. Ответ: 257
